対象バージョン: 2026-07-01 時点の AnodeHeartLayerNull_DeckBuilder

## 総評

**全体的に完成度は高い。** 3カラム構成・ビュー切替・列操作はすでに実装済みで、業界水準を満たしている。  
主な改善余地は「デッキペイン側のビュー切替不足」の1点。列順は対応済み。

## 1. レイアウト分割方向

### 現状
```
[ デッキペイン (28rem) ] [ カードプール (flex) ] [ フィルタ (18rem) ]
      左固定ペイン            中央メイン領域           右サイドバー
```

### 調査結果との比較

| ツール | 分割構成 |
|---|---|
| Limitless TCG | 左=デッキ / 右=カードプール |
| Digimon Card App | 左=デッキ / 右=カードプール |
| arkham.build | 左=デッキ / 中=カードプール / 右=フィルタ |
| **現状** | **左=デッキ / 中=カードプール / 右=フィルタ** |

### 評価: ✅ 業界標準に合致

arkham.build と同じ3カラム構成（左=デッキ / 中=カードプール / 右=フィルタ）になった。
Limitless・Digimon も「デッキが左」という点で一致しており、業界標準に沿った配置。

## 2. ビュー切替（リスト/グリッド）

### 現状
- カードプール側: ✅ リスト/グリッド切替あり（ヘッダー右の `#view-toggle` ボタン）
- デッキペイン側: ❌ グリッド（タイル表示）固定、切替なし

### 調査結果との比較
- Archidekt: デッキ側が Stacks/Text/Grid/Visual の4択
- Limitless: デッキ側にもグリッド/リスト切替あり
- Digimon Card App: カードプール側がグリッド固定（切替なし）→ これは許容範囲の事例

### 評価: ⚠️ デッキペイン側は固定でも許容されるが、追加余地あり

カードプール側にあれば最低限OK（Digimon Card App がその事例）。  
ただし Archidekt・Limitless のようにデッキ側もリスト切替ができるとカード枚数確認がしやすくなる。

## 3. 列順

### 現状の列順（`layout.js:12-14`）
```
attrClass → img → nameId → effect → cost → lv → hp → bp → bonus → type
```

### 調査結果の推奨列順
```
img → attrClass → nameId → effect → cost → lv → hp → bp → bonus → type
```

### 差分

| 列 | 現状 | 推奨 | 状態 |
|---|---|---|---|
| `effect`（特性/効果） | nameId の直後（4列目） | nameId の直後（4列目） | ✅ 一致 |
| `type`（タイプ） | **最右端（10列目）** | **最右端（10列目）** | ✅ 一致 |

### 評価: ✅ 対応済み

`layout.js` の `BASE_ORDER` がすでに推奨列順を反映している。  
`effect` は nameId の直後（4列目）、`type` は右端（10列目）となっており、調査結果との乖離なし。

## 4. フィルタUI

### 現状
- 位置: 左サイドバー（固定幅 18rem）
- 項目: タイプ / キーワード / レベル / 属性 / 種族 / カード範囲
- 保存: localStorage (`filter-defaults-v1`)、セクションごとに保存/リセット

### 調査結果との比較

| ツール | フィルタ位置 |
|---|---|
| Digimon Card App | 上部バー（コンパクト） |
| Limitless TCG | 上部バー |
| Archidekt | 上部バー |
| arkham.build | **独立右パネル（折りたたみ式、10種以上）** |
| **現状** | **独立左パネル（常時表示）** |

### 評価: ✅ 概ね良好、将来は折りたたみ検討

左パネル固定は業界の主流（上部バー）とは異なるが、フィルタ項目数が多い本プロジェクトでは
arkham.build 方式（独立パネル）のほうが適しており、それに近い設計になっている。  
将来的にフィルタ項目が増えた場合は各セクションの折りたたみ（expand/collapse）を検討。
