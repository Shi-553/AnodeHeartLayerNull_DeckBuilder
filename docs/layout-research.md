## 2. ビューモード(リスト型 vs グリッド型)

- 明示的な「表示切替」を持つのが業界標準（Archidekt・Limitless・arkham.build で確認）。
- **Digimon Card App はグリッド固定**（切替なし）。一律グリッドで問題ない設計もある。
- グリッド/カード型: 画像前面・視覚的スキャンが速い・モバイル向き。一覧性とソート/ハイライト性は落ちる。
- リスト(表)型: 情報密度が高い・列ソート/グループ化・**検索語ハイライトに強い**。
- 結論: リスト型の利点を残しつつ、グリッド型を別ビューとして追加し**トグルで切替**（Archidekt・Limitless の方式）。

## 3. 列順 / 情報階層

- カードの情報優先度は「**名前(最重要)→ 効果テキスト → コスト/キーワード等の副次情報**」。
- → 効果テキストを画像・名前の近くへ繰り上げ、数値系(コスト/Lv/HP/BP/補正/タイプ)を右へ寄せる。
- 採用デフォルト列順: `属性/種族 → 画像 → 名前/ID → 特性/効果 → コスト → Lv → HP → BP → +HP/+BP → タイプ`

## 4. 分割方向

- デスクトップツールは「**左右分割**」が主流（Limitless・Digimon で確認）。
- **Archidekt は例外**: カード検索が左上の小パネルで、デッキ表示が中央を占めるシングルパネル寄りの設計。
- **arkham.build は3カラム**: デッキ(左) + カードプール(中) + フィルタ(右)。フィルタを独立させているのが特徴。
- 縦方向の視線移動(スクロール領域↔固定領域の往復)は迷子になりやすい → 左右分割が基本方針。
- → メインを左右分割(左=検索プール / 右=デッキ編集)に変更。フィルタは将来的に独立パネル化も検討。

## 5. フィルタUIのパターン

- **上部バー型**（Digimon・Limitless）: コンパクト、常時表示
- **独立右パネル型**（arkham.build）: フィルタ項目が多い場合に有利、折りたたみ式セクションが使いやすい
- arkham.build では Type / Level / Cost / Trait / Properties(チェックボックス多数) / Pack など10種以上のフィルタを右パネルで管理
