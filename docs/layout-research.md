# レイアウト調査メモ(検索結果テーブル/デッキ編集)

レイアウト大改修にあたり、一般的なデジタルカードゲームの Web 版デッキ構築ツールを調査した結果のまとめ。

調査対象: Archidekt / Moxfield / Limitless TCG / Digimon Card App / DeckPlanet、および UI ベストプラクティス記事。

## 1. ビューモード(リスト型 vs グリッド型)

- 明示的な「表示切替」を持つのが業界標準。Archidekt は View as = Stacks / Text / **Grid**、Moxfield は text / images 切替。
- グリッド/カード型: 画像前面・視覚的スキャンが速い・モバイル向き。一覧性とソート/ハイライト性は落ちる。
- リスト(表)型: 情報密度が高い・列ソート/グループ化・**検索語ハイライトに強い**。
- 結論: リスト型の利点を残しつつ、グリッド型を別ビューとして追加し**トグルで切替**。

## 2. 列順 / 情報階層

- カードの情報優先度は「**名前(最重要)→ 効果テキスト → コスト/キーワード等の副次情報**」。
- → 効果テキストを画像・名前の近くへ繰り上げ、数値系(コスト/Lv/HP/BP/補正/タイプ)を右へ寄せる。
- 採用デフォルト列順: `属性/種族 → 画像 → 名前/ID → 特性/効果 → コスト → Lv → HP → BP → +HP/+BP → タイプ`。

## 3. 分割方向(縦 vs 左右)

- デスクトップのデッキ構築ツールは「左右分割」(片側=デッキ、残り=検索プール)が主流。
- 縦方向の視線移動(スクロール領域↔固定領域の往復)は迷子になりやすい。
- → メインを左右分割(左=検索プール / 右=デッキ編集)に変更。

## 反映先

- 列順・列幅・表示モードは「レイアウト」概念で一括し、`localStorage('layout-defaults-v1')` に保存/復元(ヘッダの 💾/↺ ボタン)。
